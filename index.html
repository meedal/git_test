CSS 가이드

기본

CSS 데이터를 로딩할 경우 @import 방식은 사용하지 않는다. (일부 브라우저에서 이미지 로딩 후 적용되기 때문에)

External 형식의 CSS 문서에는 HTML 문자셋과 동일하게 문자셋 정보를 명시한다.
영문 이외의 서체이름을 사용하는 경우 웹 브라우저에서 정확한 서체표현이 가능하고 각종 저작도구에서 문자값이 깨지지 않는다.
문자셋 선언은 문서의 첫 줄에 공백없이 삽입되어야 하고 문자셋 선언을 생략하는 것이 문법적으로 오류는 아니지만 원활한 유지보수를 위하여 반드시 포함시켜야한다.

			@charset "utf-8"

CSS 데이터는 섹션의 경우 head 상단에 공통부분(common.css)과 콘텐츠부분(해당서비스.css)으로, 간단한 콘텐츠의 경우 하나의 css 파일로 로딩될 수 있도록 한다.

CSS 선택자 기준으로 개행을하고 속성간 개행은하지 않으며, 클래스명을 선언한 뒤와 세부 속성간에는 한칸의 공백을 준다, CSS 선언의 마지막 속성의 세미콜론은 생략한다.

			.gnb {overflow:hidden; width:980px; margin:0 auto} (o)

CSS 선택자는 class로 핸들링하며, 태그네임으로 핸들링하지 않는다. (단 사용빈도가 높은 li, td, th는 예외) 또한, 태그네임과 class는 중복사용하지 않는다.

			h1 {font-size:20px; line-height:24px} (x)
			h1.tit1 {font-size:20px; line-height:24px} (x)
			.tit1 {font-size:20px; line-height:24px} (o)
			.news li {float:left; margin:0 10px 0 0} (o)
			.news li.fst (X) / .news .fst (O)

의미 있는 객체를 구분하기 위하여 코드 그룹 간 행 간격을 1줄 이상 비우지 않는다.
